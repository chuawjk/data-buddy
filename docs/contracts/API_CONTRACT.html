Section 1

REST endpoints

All endpoints are synchronous and return JSON. Agent-driven operations fire a prompt to OpenCode and return immediately — progress arrives via the SSE stream. Backend-only operations complete in the response body.

GET /state Hydrate full UI state on page load

Returns the complete current state of the brief. Called once on page load (and on page refresh). The SPA renders from this response; no further reads needed until the next page load. See Section 3 for the full response schema.

Request body: none  ·  200 OK → full state object

POST /setup Upload dataset and aim · start brief

Accepts the CSV file and the user's stated aim. Initialises state.json, creates the OpenCode session, and fires the profiling prompt. Progress arrives via SSE.

FieldTypeRequiredNotes
filemultipart/form-datayesCSV file. Saved to data/<filename>.csv in the workspace.
aimstringyesUser's stated aim of investigation. Stored in state.json and included in every subsequent prompt.

200 OK → { "ok": true, "session_id": "ses_..." }

POST /turn Bottom-bar free-form input

Routes bottom-bar text to the agent. Context-aware: the orchestrator wraps the text in stage-appropriate framing before sending to OpenCode. Used in all stages where the bottom bar is active (profiling feedback, plan revision, section redirect).

FieldTypeRequiredNotes
textstringyesThe user's raw bottom-bar input.

204 No Content · progress via SSE

POST /plan/accept Accept plan · begin section build

Backend-only. Transitions state.stage to "building", writes updated state.json, and fires the first section's build prompt. Returns immediately; section build progress via SSE.

Request body: none  ·  204 No Content

POST /plan/update Inline plan edit (backend-only)

Backend-only. Accepts a full replacement plan array from the SPA (used for inline edits, reorder, drop, add). Writes directly to plan.json and state.json. No OpenCode involvement.

FieldTypeRequiredNotes
sectionsarrayyesFull replacement plan array. Each item: { id, title, hypothesis, status }. Status must be "queued" for all items at plan stage.

200 OK → { "ok": true }

POST /section/:id/accept Accept proposed section · trigger next

Backend-only for the state update; triggers an agent turn for the next section. Flips the section's status from "proposed" to "accepted" in state.json. If more sections remain, immediately fires the next section's build prompt via OpenCode. If this was the last section, transitions state.stage to "done".

Path param: id — section ID, e.g. sec_02  ·  204 No Content

POST /section/:id/drop Drop proposed section · trigger next

Backend-only. Deletes the section's three workspace files (analyses/, charts/, sections/), removes the section from the plan array in state.json, and fires the next section's build prompt. If no sections remain, transitions to "done".

Path param: id — section ID  ·  204 No Content

GET /file Serve workspace file (code / chart)

Serves a file from the workspace directory. Used by the SPA to load chart PNGs and analysis scripts for display in the section pane. Restricted to the workspace directory; path traversal rejected.

ParamTypeRequiredNotes
pathquery stringyesRelative path within the workspace, e.g. charts/sec_02_churn_by_tier.png. Must not contain ...

200 OK → file bytes with correct Content-Type

GET /export Export brief as Markdown

Backend-only. Concatenates all accepted section files from sections/ in plan order into a single Markdown document with a title block. Returns as a file download. Available from Stage 4 onward.

200 OK → Content-Disposition: attachment; filename="brief.md"

Section 2

SSE event catalogue

The backend emits a single SSE stream at GET /events. These are backend-enriched events — derived from OpenCode's raw event stream but normalised, filtered to the current session, and augmented with stage context. The SPA subscribes once on mount and never reconnects unless the connection drops.

Wire format: standard SSE. Each event: event: <type>\ndata: <json>\n\n.

Activity events

Rendered in the right-rail activity stream. Derived from OpenCode's message.part.updated tool events.

tool.file_written Agent wrote a file via apply_patch

Source: OpenCode message.part.updated where part.tool === "apply_patch" and part.state.status === "completed"

{ "type": "tool.file_written", "file": "analyses/sec_02_churn_by_tier.py", // relativePath from metadata.files[0] "op": "add", // "add" | "modify" | "delete" "additions": 42, "deletions": 0, "elapsed_ms": 5, // state.time.end - state.time.start "ts": 1779784985520 }

Activity rail: "Wrote analyses/sec_02_churn_by_tier.py" with additions count. If the file is a .py, section pane updates the code display.

tool.bash_running Agent started a bash command

Source: OpenCode message.part.updated where part.tool === "bash" and part.state.status === "running"

{ "type": "tool.bash_running", "command": "python analyses/sec_02_churn_by_tier.py", // state.input.command "description": "Run churn by tier analysis", // state.input.description "started_at": 1779784977098, "ts": 1779784977098 }

Activity rail: "Running sec_02_churn_by_tier.py" with a live elapsed timer (frontend starts counting from started_at).

tool.bash_done Bash command completed

Source: OpenCode message.part.updated where part.tool === "bash" and part.state.status === "completed"

{ "type": "tool.bash_done", "command": "python analyses/sec_02_churn_by_tier.py", "exit_code": 0, "elapsed_ms": 1203, "ts": 1779784978301 }

Activity rail: marks running item as done with elapsed time. Non-zero exit_code renders as an error line (red !) but does not surface a failure banner — agent may self-recover.

Section lifecycle events

Drive section pane state and Accept/Drop button visibility.

section.building Backend started building a section

Source: Backend-generated (emitted immediately after firing the section build prompt to OpenCode)

{ "type": "section.building", "section_id": "sec_02", "title": "Churn by plan tier", "ts": 1779784970000 }

Stage view: advances section status indicator in left rail to "building". Section pane: clears previous content, shows spinner.

section.proposed Section complete · awaiting user review

Source: Backend-generated on session.idle after verifying all three workspace files exist for the section

{ "type": "section.proposed", "section_id": "sec_02", "py_path": "analyses/sec_02_churn_by_tier.py", "png_path": "charts/sec_02_churn_by_tier.png", "md_path": "sections/sec_02_churn_by_tier.md", "ts": 1779784990000 }

Section pane: surfaces Accept / Drop buttons. Activity rail: appends "Section ready for review" marker. SPA loads files via GET /file?path=....

section.failed Section build failed · retry available

Source: Backend-generated on session.idle when expected workspace files are absent, or on watchdog timeout after abort

{ "type": "section.failed", "section_id": "sec_02", "reason": "timeout", // "timeout" | "output_error" | "missing_files" "ts": 1779784995000 }

Section pane: surfaces retry banner — "The agent couldn't complete this section." Buttons: Retry (re-fires section prompt), Drop (removes from plan, advances).

Stage transition events

Drive top-level stage view switching in the SPA.

stage.changed Orchestrator moved to a new stage

Source: Backend-generated on every state.stage write

{ "type": "stage.changed", "stage": "building", // "profiling" | "planning" | "building" | "done" "ts": 1779784970000 }

Stage view: switches rendered view. SPA does NOT call GET /state again — it trusts the event and updates the stage indicator only. Full hydration only on page load.

profile.ready Profile structured output parsed and written

Source: Backend-generated after parsing the profiling turn's structured output and writing profile.json

{ "type": "profile.ready", "profile": { /* full profile object — see state schema §3 */ }, "ts": 1779784960000 }

Profile view: renders shape strip and column rows directly from this payload. No additional fetch needed.

plan.ready Plan structured output parsed and written

Source: Backend-generated after parsing the planning turn's structured output and writing plan.json

{ "type": "plan.ready", "sections": [ { "id": "sec_01", "title": "Cohort overview", "hypothesis": "...", "status": "queued" }, // … ], "ts": 1779784965000 }

Plan view: renders editable section list directly from this payload.

Error and liveness events

turn.error Agent turn failed · retry banner

Source: Backend-generated on structured output failure, watchdog timeout, or provider error

{ "type": "turn.error", "stage": "profiling", // which stage failed "reason": "structured_output_failed", // "structured_output_failed" | "timeout" | "provider_error" "ts": 1779784970000 }

Stage view: surfaces inline retry banner "Couldn't complete this step — retry." Retry button calls POST /turn with an empty body (re-fires the last prompt).

heartbeat Keep-alive · connection healthy

Source: Backend-generated every 15s, proxied from OpenCode's server.heartbeat

{ "type": "heartbeat", "ts": 1779784980000 }

SPA: resets its own connection-health timer. If no heartbeat for 30s, display reconnecting indicator and call EventSource.close(); new EventSource('/events').

Section 3

GET /state — full response schema

Complete structure of the GET /state response. Mirrors state.json exactly, minus the internal opencode_session_id field (not exposed to the SPA).

{ "version": "1", "stage": "building", // "setup"|"profiling"|"planning"|"building"|"done" "aim": "Understand drivers of customer churn in Q3 2025", "dataset_path": "data/customers_q3.csv", "last_saved": "2026-05-30T11:08:42Z", "profile": { "shape": { "rows": 12450, "columns": 9, "nulls_pct": 0.2, "target": "churned" // null if not inferred }, "columns": [ { "name": "plan_tier", "type": "cat", // "str"|"int"|"num"|"cat"|"date"|"bool" "summary": "basic 58% · pro 31% · enterprise 11%", "nulls_pct": 0.0, "flags": [] } // … one entry per column ], "flags": [] // dataset-level concerns }, // null until profiling complete "plan": [ { "id": "sec_01", "title": "Cohort overview & churn baseline", "hypothesis": "Establish the Q3 churn rate and frame the brief.", "status": "accepted", // "queued"|"building"|"proposed"|"accepted"|"dropped"|"failed" "py_path": "analyses/sec_01_cohort_overview.py", // null until built "png_path": "charts/sec_01_cohort_overview.png", "md_path": "sections/sec_01_cohort_overview.md" } // … one entry per plan section ] // [] until planning complete }

The SPA should treat profile: null and plan: [] as valid states — both occur during early stages. The stage field is the primary switch for which view to render.

Section 4

Error responses

All error responses share a consistent envelope.

{ "error": "section_not_found", // machine-readable code "message": "No section with id sec_99 exists in the current plan." }
StatusError codeWhen
400invalid_stageAction is not valid for the current stage (e.g. /section/:id/accept called during planning).
400section_not_foundSection ID does not exist in the current plan.
400section_not_proposedAccept or drop called on a section that is not in proposed status.
400turn_busyA /turn request arrived while another agent turn is in flight. SPA should disable the bottom bar while session.idle is pending.
400missing_fileFile requested via GET /file does not exist in the workspace.
400path_traversalFile path in GET /file contains .. or escapes the workspace root.
