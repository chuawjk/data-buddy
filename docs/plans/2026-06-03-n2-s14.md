## What this does

Replace the stub `GET /file` handler in `backend/router.py` with a real implementation that serves
files from the `workspace/` directory with correct content-type, rejects path traversal, and
returns a 404 + error envelope for missing files.

## Approach

### Handler logic (in `router.py`)

1. Read the `path` query parameter (already declared with `Annotated[str, Query(...)]` in the stub).
2. Reject immediately if `path` is empty or contains traversal sequences тАФ check for `..` components
   after resolving: compute `(workspace_root / path).resolve()` and verify it is contained within
   `workspace_root.resolve()`. Return 403 with error envelope `{"error": "forbidden", "message": "..."}`.
3. Compute the absolute path: `workspace_root / path`.
4. If the file does not exist, return 404 with error envelope `{"error": "missing_file", "message": "..."}`.
5. Determine content-type from the file suffix using Python's `mimetypes.guess_type()`, with
   explicit overrides for common types:
   - `.png` тЖТ `image/png`
   - `.py`  тЖТ `text/x-python`
   - `.md`  тЖТ `text/markdown`
   - `.csv` тЖТ `text/csv`
   - `.json` тЖТ `application/json`
   Fallback to `application/octet-stream` for unknown types.
6. Return `FileResponse(path=abs_path, media_type=content_type)` тАФ FastAPI's `FileResponse`
   handles streaming and sets `Content-Length`; this is safe for binary files (PNGs).

### Workspace root resolution

The handler needs the workspace root. The cleanest approach (consistent with how `POST /setup`
already does it) is to derive it from `request.app.state.state_manager._path.parent`. This avoids
a new config entry and keeps the handler aligned with the rest of the app.

### Path traversal check

Canonical check: after calling `.resolve()` on both the candidate path and the workspace root,
verify that the candidate starts with the workspace root (using `str(candidate).startswith(str(root))`
or equivalently `candidate == root or root in candidate.parents`). Any path that escapes the
workspace root gets a 403 before the filesystem is touched.

### Error envelopes

Both error responses use the same JSON structure already established in the codebase:
```json
{"error": "<code>", "message": "<human-readable>"}
```
This matches `API_CONTRACT.html` ┬з4 error envelope shape.

## Test plan (TDD тАФ write tests first, then implementation)

All tests in `backend/tests/unit/app/test_file_endpoint.py`.

**Happy path:**
- `GET /file?path=data/test.csv` with file present тЖТ 200, correct media type `text/csv`
- `GET /file?path=charts/sec_01_test.png` with PNG present тЖТ 200, `image/png`
- `GET /file?path=sections/sec_01_test.md` with markdown present тЖТ 200, `text/markdown`

**Error paths:**
- Missing file тЖТ 404, error envelope with `"error": "missing_file"`
- Path traversal `../../etc/passwd` тЖТ 403 (or 400), error envelope with `"error": "forbidden"`
- Absolute path `/etc/passwd` тЖТ 403, error envelope
- Path with embedded traversal `data/../../../etc/passwd` тЖТ 403

**Edge cases:**
- Empty `path` parameter тЖТ 400 or 403 (treat as forbidden/invalid)
- Unknown file extension тЖТ 200, `application/octet-stream`
- Zero-byte file тЖТ 200 (FileResponse handles gracefully)

**Null / missing inputs:**
- Missing `path` query param тЖТ FastAPI raises 422 automatically (no explicit test needed; covered
  by FastAPI's own validation, but one test confirms the 422 shape)

## Risks

1. **`FileResponse` import** тАФ `fastapi.responses.FileResponse` is in FastAPI stdlib; no new
   dependency required.
2. **Symlink traversal** тАФ `.resolve()` dereferences symlinks; a symlink inside `workspace/`
   pointing outside is caught by the containment check. No special symlink handling needed.
3. **Test isolation** тАФ tests need a real temp filesystem (use `tmp_path` fixture). `FileResponse`
   reads from disk, so we cannot mock it away without also testing less. Use `TestClient` +
   `tmp_path`-based workspace override.
4. **Workspace root from `state_manager._path`** тАФ this is an internal attribute; it is already
   used by `POST /setup` handler (`workspace_root: Path = state_manager._path.parent`) so the
   pattern is established. No new coupling introduced.
