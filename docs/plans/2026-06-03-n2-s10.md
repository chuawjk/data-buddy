**Date:** 2026-06-03

## What this does

Implements `POST /section/:id/accept` — marks a proposed section as accepted in
`state.json` and returns 204. No OpenCode call is made.

## Files to create/modify

- `backend/router.py` — replace the stub `post_section_accept` handler with real logic
- `backend/tests/unit/app/test_section_accept.py` — new test file (TDD)

## Approach

1. Read `state_manager.get_state()["plan"]` to find the section by `section_id`.
2. If not found → 400 `section_not_found` error envelope.
3. If found but status is not `"proposed"` → 400 `section_not_proposed` (contract §4).
4. Set `status = "accepted"` on the matching section.
5. `state_manager.update(plan=updated_plan)`.
6. Return 204.

No OpenCode call, no bus event for this story. The router reads from
`request.app.state.state_manager` (same pattern as `/export`).

## Risk

None — purely synchronous state mutation. The `state_manager.update()` call is
already used throughout the codebase.
