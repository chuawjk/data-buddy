## What this does

Implements `POST /plan/accept` — locks the plan, transitions stage from
`planning` to `building`, emits `stage.changed`, triggers the first
proposed section's build prompt via the narrow client interface, returns 204.

## Files to create/modify

- `backend/router.py` — replace the stub `post_plan_accept` handler, call orchestrator
- `backend/orchestrator.py` — add `accept_plan()` method
- `backend/tests/unit/app/test_accept_plan.py` — new test file (TDD)

## Approach

### Router changes

`post_plan_accept` becomes:
```python
async def post_plan_accept(request: Request) -> Response:
    asyncio.create_task(request.app.state.orchestrator.accept_plan())
    return Response(status_code=204)
```

The router immediately returns 204; `accept_plan` runs as fire-and-forget.

### Orchestrator.accept_plan()

1. Read stage from state. If stage is already `"building"` → return silently (idempotent).
2. If stage is not `"planning"` → log warning and return (guard).
3. `state_manager.update(stage="building")`.
4. Emit `stage.changed` with `{"stage": "building"}` on bus.
5. Find the first section in plan with status `"proposed"`.
6. If no such section exists, log and return (degenerate empty plan).
7. Call `build_section_prompt()` from `backend.prompts.section` to build the prompt text.
8. Arm watchdog if present (`watchdog.start_turn()`).
9. `asyncio.create_task(_run_section_turn(session_id, prompt_text))`.

### _run_section_turn helper

Similar to `_run_profile_turn`:
- Calls `client.prompt(session_id, text, schema=None)` (ADR-005: no schema for sections).
- Catches exceptions → publishes `turn.error`.

## Architecture constraints

- Orchestrator never imports httpx (verified by existing test).
- `accept_plan()` only calls `client.prompt()` — the narrow interface.

## Idempotency

If `accept_plan()` is called when stage is already `"building"`, return immediately.
The test verifies repeated calls do not re-trigger the section build.
