**Date:** 2026-06-04
**Stories:** N3-S02 (retry a turn), N3-S03 (map turn errors), N3-S16 (forced turn-error hook)
**Branch:** feat/n3-s02-s03-s16-retry-turn-errors

---

## What these stories do

**N3-S02 ‚Äî Retry a turn**
The retry button calls `POST /turn` with an empty body to re-fire the last prompt.
The orchestrator must remember what it last dispatched and replay it, bounded to 3 retries.

**N3-S03 ‚Äî Map turn errors**
`turn.error` events currently only carry `message`. The API contract requires `stage`,
`section_id` (building only), and `retryable` fields. All three `_run_*_turn` methods
need to emit the full payload.

**N3-S16 ‚Äî Forced turn-error hook**
A `QA_FORCE_TURN_ERROR=1` env-var seam makes `client.prompt()` raise before any OpenCode
traffic, allowing QA to exercise the `turn.error` ‚Üí retry banner path without token spend.

## Implementation plan

### 1. N3-S16 first: `QA_FORCE_TURN_ERROR` seam in `opencode_client.py`

Place the check at the top of `OpenCodeClient.prompt()`, before the httpx call:

```python
if os.environ.get("QA_FORCE_TURN_ERROR") == "1":
    raise RuntimeError("QA_FORCE_TURN_ERROR: simulated turn error")
```

Update the module docstring to note the seam.
This is a one-liner, cleanest in the single `prompt()` call-site.

### 2. N3-S03: Add `stage`, `section_id`, `retryable` to `turn.error` payloads

Update `_run_profile_turn`, `_run_plan_turn`, and `_run_section_turn` in `orchestrator.py`:

- `_run_profile_turn`: emit `{"stage": "profiling", "retryable": True, "message": str(exc)}`
- `_run_plan_turn`: emit `{"stage": "planning", "retryable": True, "message": str(exc)}`
- `_run_section_turn`: needs `section_id` threaded in as a parameter; emit
  `{"stage": "building", "section_id": section_id, "retryable": True, "message": str(exc)}`

Thread `section_id` through the callers:
- `start_build_section(section_id, ...)` already has it ‚Äî pass it to `_run_section_turn`
- `redirect_section(...)` already has `section_id` ‚Äî pass it to `_run_section_turn`

### 3. N3-S02: `_last_turn` field + `retry_last_turn()` on `Orchestrator`

Add to `__init__`:
```python
self._last_turn: dict | None = None
```

At the start of each `_run_*_turn`, record:
```python
self._last_turn = {
    "stage": "profiling",   # or "planning" / "building"
    "prompt": prompt,
    "section_id": None,     # or the actual section_id for building
    "retries": self._last_turn.get("retries", 0) if self._last_turn else 0,
}
```

After a successful `client.prompt()` call, reset retries:
```python
if self._last_turn:
    self._last_turn["retries"] = 0
```

Add `retry_last_turn()`:
- No `_last_turn` ‚Üí log warning, return
- `retries >= 3` ‚Üí publish `turn.error` with `retryable=False`, return
- Increment retries, re-dispatch via the right `_run_*_turn` method
- Session ID read fresh from `state_manager.get_state()["opencode_session_id"]`

### 4. Update `POST /turn` in `router.py`

Change the body parameter to be optional (`Body(default=None)`).
If body is absent or `text` is absent/empty: call `orchestrator.retry_last_turn()`, return 204.
Otherwise route as before.

## Test plan

All tests in `backend/tests/unit/core/test_orchestrator.py`:

1. `test_retry_last_turn_replays_profile_prompt` ‚Äî after a profiling turn records `_last_turn`,
   `retry_last_turn()` calls `client.prompt` again with the same prompt.
2. `test_retry_last_turn_uses_fresh_session` ‚Äî simulate watchdog session swap (update
   state session_id), verify retry uses the new session ID.
3. `test_retry_bounded_at_three` ‚Äî call retry 4 times; 4th emits `turn.error` with
   `retryable=False` and does not call `client.prompt` again.
4. `test_retry_without_prior_turn` ‚Äî `retry_last_turn()` with `_last_turn=None` returns
   without error or `client.prompt` call.
5. `test_turn_error_has_stage_and_retryable` ‚Äî `_run_profile_turn` exception emits a
   `turn.error` payload with `stage="profiling"` and `retryable=True`.
6. `test_qa_force_turn_error` ‚Äî with env var set, `client.prompt()` raises and `turn.error`
   is emitted via `_run_profile_turn`.

Router test in `test_turn.py`:
- `test_turn_empty_body_calls_retry` ‚Äî `POST /turn` with `{}` calls `retry_last_turn()`.
