# Plan: N2-S13 — Export brief (GET /export)

**Story:** N2-S13  
**Branch:** feat/n2-s13-export-brief  
**Date:** 2026-06-03  
**Role:** BE  
**Size:** S

## Problem statement

`GET /export` currently returns a stub. The real implementation must concatenate
the bodies of all accepted sections in plan order into a single Markdown file
download, using `Content-Disposition: attachment; filename="brief.md"`.

## Approach

Replace the stub in `backend/router.py` `get_export()` with real logic:

### 1. Read the plan from state

```python
state_manager = request.app.state.state_manager
state = state_manager.get_state()
plan_sections = state.get("plan", [])
```

### 2. Filter and order accepted sections

Only sections with `status == "accepted"` are included; `"dropped"`,
`"proposed"`, `"building"`, `"failed"` sections are excluded. The plan list is
already in plan order (the order the user saw and accepted).

### 3. Find the .md file for each accepted section

Section IDs are like `sec_01`, `sec_02`. The workspace section files are named
`sections/sec_NN_<slug>.md` (e.g. `sections/sec_01_churn_overview.md`). The
backend resolves this via a glob:
```python
workspace_root / "sections" / f"{section_id}_*.md"
```

If a section has an `md_path` stored (set by N2-S07 after building), use that
directly; fall back to glob if absent or if the stored path doesn't exist.

### 4. Parse each file and extract the body

Use `frontmatter_parser.parse_section_file()` which returns `{"body": ..., ...}`.
The body is the content after the frontmatter block.

### 5. Concatenate with separator

Join bodies with `\n\n---\n\n` between sections.

### 6. Return the Markdown response

```python
PlainTextResponse(
    content=combined,
    media_type="text/markdown",
    headers={"Content-Disposition": 'attachment; filename="brief.md"'},
)
```

### 7. No accepted sections fallback

When no sections are accepted, return the default markdown:
```
# Brief\n\n*(no accepted sections yet)*\n
```

### 8. Helper: find section file

Add a private helper `_find_section_file(workspace_root, section)`:
- If section has `md_path` (not None), try `workspace_root / section["md_path"]`.
- Otherwise glob `workspace_root / "sections" / f"{section['id']}_*.md"`.
- Returns the first match or `None` if not found.

## Changes

**`backend/router.py`:**
- Replace stub `get_export()` with real implementation.
- Add `_find_section_file()` helper.
- Import `parse_section_file` from `backend.frontmatter_parser`.
- Signature change: `get_export(request: Request)` to access state_manager.

## Tests (new file: `backend/tests/unit/app/test_export.py`)

1. **`test_export_accepted_sections_concatenated`** — happy path: 2 accepted
   sections → bodies concatenated in plan order with separator.
2. **`test_export_dropped_sections_excluded`** — dropped section excluded.
3. **`test_export_proposed_sections_excluded`** — proposed (not yet accepted) excluded.
4. **`test_export_no_accepted_sections_default_markdown`** — empty plan →
   default "no accepted sections yet" markdown.
5. **`test_export_plan_order_preserved`** — sections appear in plan.json order,
   not file-system order.
6. **`test_export_uses_frontmatter_parser`** — body is extracted after
   frontmatter (not the raw file content).
7. **`test_export_zero_opencode_calls`** — no OpenCode client calls during export.
8. **`test_export_content_disposition_header`** — response has correct
   `Content-Disposition: attachment; filename="brief.md"` header.
9. **`test_export_media_type_text_markdown`** — response `Content-Type` is
   `text/markdown`.
10. **`test_export_section_with_missing_md_file_skipped`** — if the .md file
    cannot be found for an accepted section, that section is skipped gracefully.
