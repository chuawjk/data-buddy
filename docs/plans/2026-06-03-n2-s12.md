# Plan: N2-S12 — Redirect a section (Stage 4b)

**Date:** 2026-06-03
**Branch:** feat/n2-s12-redirect-section
**Story:** N2-S12 — Redirect a section (Stage 4b)

## What

The mid-section recovery moment: the user submits bottom-bar text while a section is
building; the agent discards its draft artefacts and rebuilds from scratch applying
the user's instruction.

Three parts:

### Part A — `backend/prompts/redirect.py` (new file)

`build_redirect_prompt(section_id, section_index, title, hypothesis, aim, dataset,
profile, plan, redirect_text, workspace_root)` → str

Tells OpenCode to:
1. Delete the prior draft artefacts for this section (the agent should remove
   `analyses/sec_NN_*.py`, `charts/sec_NN_*.png`, `sections/sec_NN_*.md`).
2. Rebuild from scratch per the user's redirect instruction.
3. Use the same file naming convention as `prompts/section.py`.

No structured output (no `schema=` argument) — same as section build (ADR-005).

### Part B — `backend/orchestrator.py`

`redirect_section(text)` method:
- Validates stage=building and session_id set.
- Finds the active section (first with status=building in plan).
- Deletes the prior draft files from disk before dispatching the new prompt
  (ensures clean state even if agent misses the delete instruction).
- Builds redirect prompt via `build_redirect_prompt()`.
- Arms watchdog.
- Fire-and-forgets `_run_section_turn()`.

### Part C — `backend/router.py`

`POST /turn` building-stage dispatch: calls `orchestrator.redirect_section(text)`.

2. Prior draft files deleted before prompt dispatch
3. Watchdog armed during redirect

6. `redirect_section` raises ValueError when stage != building
7. `redirect_section` no-op if no building section in plan (logs and returns)
8. Prompt contains redirect text and section context
9. client.prompt called with correct session and no schema
