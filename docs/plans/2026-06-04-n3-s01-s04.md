## What this does

N3-S01 closes the auto-sequence loop: when the last section is terminal (all sections are
`accepted`, `dropped`, or `failed` and none are `queued` or `building`), the stage transitions
to `"done"` and `stage.changed(done)` is emitted.

N3-S04 fixes a watchdog heartbeat bug: `heartbeat()` currently calls `start_turn()` with no
argument, so it always resets to the 60s default instead of preserving the 180s timeout that was
active when the section build started.

## Files to change

### Implementation

1. `backend/core/orchestrator.py`
   - Add `_check_done_or_next(section_id: str)` method:
     - Reads plan from state.
     - If any section is `"queued"` or `"building"`: no-op (auto-sequence is still running).
     - If ALL sections are terminal (`accepted`/`dropped`/`failed`) AND stage is `"building"`:
       persist `stage="done"`, emit `stage.changed(done)`.
     - Re-entrant guard: double-check stage inside the method before emitting.
   - Update `_start_next_queued_section`: when no queued sections are found, call
     `await self._check_done_or_next("")` as a belt-and-suspenders fallback.

2. `backend/api/router.py`
   - Update `POST /section/:id/accept`: after persisting `status="accepted"`, schedule
     `asyncio.create_task(orchestrator._check_done_or_next(section_id))`.
   - Update `POST /section/:id/drop`: same pattern for `status="dropped"`.

3. `backend/core/watchdog.py`
   - Store `_current_timeout` when `start_turn()` is called.
   - `heartbeat()` calls `self.start_turn(self._current_timeout)` instead of `self.start_turn()`.

## Approach

### _check_done_or_next

Terminal statuses: `{"accepted", "dropped", "failed"}`.

The method:
1. Reads state, checks stage == "building" (guard).
2. Computes non-terminal = plan statuses minus terminal set.
3. If non-terminal is non-empty: return (loop still running).
4. All terminal: persist stage="done", emit stage.changed(done).

Re-entrant safety: the stage guard prevents double-emit because after the first call persists
`stage="done"`, any concurrent second call reads `stage="done"` and returns early.

### router.py changes

After the `state_manager.update(plan=updated_plan)` call in each section endpoint, add:
```python
asyncio.create_task(
    request.app.state.orchestrator._check_done_or_next(section_id)
)
```
This is fire-and-forget so the 204 response is not delayed.

### watchdog.py fix

Store the timeout when start_turn is called. heartbeat re-uses stored timeout.

---

## Risks

- **Race between two rapid accepts**: handled by stage guard inside `_check_done_or_next` -
  stage changes from "building" to "done" after first completion, making all subsequent calls no-ops.
- **`_start_next_queued_section` belt-and-suspenders**: when the auto-sequence path (session.idle)
  triggers all sections through to terminal, calling `_check_done_or_next` there also closes the
  loop correctly.
- **Watchdog fix is backward-compatible**: `_current_timeout` attribute initialised via `getattr`
  default so existing tests that call `heartbeat()` without prior `start_turn()` do not break.
