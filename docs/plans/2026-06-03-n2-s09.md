## What this does

Implement a deterministic parser that separates YAML frontmatter from the body of
`sections/*.md` files. Malformed frontmatter is handled fail-safe (section flagged, no crash).
Parsed output is consumed by `GET /state` (section metadata) and `GET /export` (ordering,
body concatenation).

## Approach

### Frontmatter format (from WORKFLOW_ORCHESTRATION.md §Stage 4)

OpenCode writes sections as:

```markdown
---
chart: charts/sec_01_churn_by_tier.png
section_id: sec_01
---

# Section Title

Body text...
```

The frontmatter is delimited by `---` on its own line at the start of the file and another `---`
line to close it. Content between the delimiters is YAML key-value pairs.

### Parser module: `backend/frontmatter_parser.py`

Expose two public functions:

#### `parse_frontmatter(text: str) -> tuple[dict, str]`

Returns `(frontmatter_dict, body_text)`.

- If the text starts with `---\n` (or `---\r\n`), scan for the closing `---` line.
- Parse the content between delimiters as YAML using `yaml.safe_load()` (PyYAML is already a
  transitive dependency via httpx/pydantic; if not, use the stdlib-safe manual key:value split
  as a fallback — no new hard dependency).
- If YAML parsing raises an exception, return `({}, full_text_as_body)` — fail safe.
- If no frontmatter block is found, return `({}, full_text)` — treat entire file as body.
- Strips leading newlines from the body after the closing delimiter.

#### `parse_section_file(path: Path) -> dict`

Reads a `sections/*.md` file from disk and returns a structured dict:

```python
{
    "path": str(path),
    "frontmatter": {...},     # parsed dict; {} if absent/malformed
    "body": "...",            # markdown body text
    "parse_error": bool,      # True if frontmatter was present but malformed
}
```

This is the shape consumed by the export builder (N2-S13) and by state hydration.

### YAML handling

Use `yaml.safe_load()`. PyYAML is already in the Python standard library ecosystem; check
`pyproject.toml` — if not present, add `pyyaml>=6.0` to `[project.dependencies]`. If PyYAML is
unavailable, implement a minimal `key: value` parser that handles the expected frontmatter fields
without a full YAML library (acceptable given the constrained schema the agent writes).

The agent-written frontmatter schema (from WORKFLOW_ORCHESTRATION.md §Stage 4) will contain
simple scalar string values only — `chart:`, `section_id:`, optionally `title:`. No nested
structures. A fallback line-by-line parser is feasible without PyYAML.

### Fail-safe semantics

The story requirement "fails safe — flag section, not crash" means:

1. A file that cannot be read (OSError) → `parse_section_file` returns `{"parse_error": True, ...}` with empty frontmatter and empty body; does not raise.
2. A file with a `---` open delimiter but no closing `---` → treat entire content as body, `parse_error: True`.
3. A file with malformed YAML between the delimiters → `frontmatter: {}`, body = everything after the opener, `parse_error: True`.
4. A completely empty file → `frontmatter: {}`, body = `""`, `parse_error: False`.

## Test plan (TDD — write tests first, then implementation)

All tests in `backend/tests/unit/app/test_frontmatter_parser.py`.

**Happy path:**
- `parse_frontmatter` on well-formed frontmatter + body → correct dict and body
- `parse_frontmatter` on file with multiple frontmatter fields → all keys parsed
- `parse_section_file` on a real temp file → correct structured dict

**Error paths:**
- Malformed YAML between `---` delimiters → `parse_error: True`, `frontmatter: {}`
- File that cannot be read (non-existent path) → `parse_error: True`, no exception raised
- Opening `---` with no closing delimiter → `parse_error: True`, body = full content

**Edge cases:**
- No frontmatter at all (plain markdown) → `frontmatter: {}`, `body = full text`, `parse_error: False`
- Empty file → `frontmatter: {}`, `body: ""`, `parse_error: False`
- Frontmatter with no body (file ends after `---`) → `frontmatter: {...}`, `body: ""`, `parse_error: False`
- CRLF line endings in frontmatter block → parsed correctly
- Leading/trailing whitespace on frontmatter values → stripped (yaml.safe_load handles this)

**Null / missing inputs:**
- `parse_frontmatter("")` → `({}, "")`, no exception
- `parse_frontmatter(None)` — type guard; should raise `TypeError` or handle gracefully (document choice)

## Risks

1. **PyYAML availability** — check `pyproject.toml` before implementing. If not present, add
   `pyyaml>=6.0`. Alternatively, since frontmatter is constrained to simple `key: value` pairs,
   a regex/split-based fallback avoids the dependency entirely. Decide at implementation time
   based on what's already in the lockfile.

2. **Frontmatter schema drift** — the agent may write frontmatter fields not in the spec. The
   parser returns the full parsed dict; callers extract known fields. No schema enforcement in
   the parser itself.

3. **Encoding** — all files written by OpenCode are UTF-8. The parser should open files with
   `encoding="utf-8"` and handle `UnicodeDecodeError` as a parse error (fail safe).
