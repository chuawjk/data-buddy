## Intent

`QA_FORCE_SECTION_FAIL=1` env-var seam in `orchestrator.py` — no model

`opencode_client.py` (N1-S20).

## Approach

### The seam

In `_handle_section_idle()` in `orchestrator.py`, before the triplet-check
block, add:

```python
if os.environ.get("QA_FORCE_SECTION_FAIL") == "1":
    md_path = self._workspace_root / "sections" / f"{base_name}.md"
    if md_path.exists():
        md_path.unlink()
    logger.info("QA_FORCE_SECTION_FAIL: deleted .md for %s to force section.failed", section_id)
```

The `.md` file is removed (or was never written). The existing triplet check
immediately after detects the missing `.md` and emits `section.failed` through
the normal code path. Zero impact on any non-hook code path.

**Why this seam location?**
- `_handle_section_idle()` is the sole decision point for pass/fail on a section
  build — already established by N2-S07/N2-S08.
- Deleting the `.md` drives the existing `missing` list logic naturally; no new
  `if` branches for the emitting path.
- The env var is only read at the seam point; the rest of the function is
  unchanged.

**Why the `.md` specifically?**
- The `.md` is the most meaningful "output" artefact — it is what the SPA reads
  and what export uses. Removing it is the minimal, cleanest forced-failure.
- Consistent with the story spec: "delete (or skip creation of) the `.md` file
  so the normal triplet check fails".

### Architecture boundary

- `orchestrator.py` reads `os.environ` (already imported). No new imports.
- No `httpx` import introduced. Boundary stays clean.

## TDD steps

1. Write `backend/tests/unit/app/test_section_fail_hook.py` with failing tests:
   - `test_force_fail_env_set_emits_section_failed`: with `QA_FORCE_SECTION_FAIL=1`,
     full triplet on disk, `_handle_section_idle()` emits `section.failed`.
   - `test_force_fail_env_set_removes_md_file`: same setup, `.md` is absent from
     disk after the call.
   - `test_force_fail_env_unset_behaviour_unchanged_proposed`: env var unset,
     full triplet present -> `section.proposed` (regression guard).
   - `test_force_fail_env_zero_behaviour_unchanged_proposed`: `QA_FORCE_SECTION_FAIL=0`,
     full triplet present -> `section.proposed`.
   - `test_force_fail_with_no_md_present_still_emits_failed`: env var set, `.md`
     already absent (e.g. OpenCode never wrote it) -> `section.failed` (no crash).
   - `test_force_fail_section_id_in_payload`: payload includes `section_id`.

2. Run tests -- expect failures.
3. Add seam to `orchestrator.py`.
4. Tests pass.
5. Run `make test` -- confirm full suite green.
